基本使用流程

  1. 登录系统（默认账号：admin，密码：admin）
  2. 在左侧功能区选择"添加新流"或"批量添加"
  3. 填写流配置信息
  4. 点击确认添加
  5. 在流列表中监控状态

添加新流

添加单个流需要填写以下信息：

{ "名称": "流的名称", "源地址": "源视频流的RTMP地址", "推流地址": "目标服务器的RTMP地址", "推流密钥": "用于验证的密钥", "转码配置": "可选择预设或自定义" }
提示：源地址支持RTMP、HTTP-FLV等多种格式

批量导入

支持以下两种方式批量导入：

  1. 通过文本框粘贴多个源地址（每行一个）
  2. 上传JSON格式的配置文件
[ { "id": "stream1", "sourceUrl": "rtmp://source1.com/live/stream1", "key": "key1" }, { "id": "stream2", "sourceUrl": "rtmp://source2.com/live/stream2", "key": "key2" } ]

转码配置

系统提供以下预设配置：

  • 原画配置：直接复制音视频流，不进行转码
  • 标清配置：480P，1000kbps视频码率
  • 高清配置：720P，2500kbps视频码率
  • 超清配置：1080P，4000kbps视频码率
提示：可以在配置管理中添加自定义转码配置

批量操作

支持的批量操作：

  • 批量启动：同时启动多个选中的流
  • 批量停止：同时停止多个选中的流
  • 批量删除：同时删除多个选中的流
注意：批量操作前请确认选中的流状态

监控和统计

系统提供以下监控指标：

  • CPU使用率
  • 内存使用情况
  • 网络带宽使用
  • 各状态流的数量统计
  • 单个流的详细性能指标

常见问题

  • 流无法启动：检查源地址是否可访问，网络是否正常
  • 转码失败：检查转码参数是否合适，服务器性能是否足够
  • 推流中断：检查网络状态，源是否正常
  • 性能问题：适当调整转码参数，减少同时转码的流数量
