1. 系统介绍

流媒体管理系统是一个功能强大的直播流管理平台，支持多种流媒体协议，可以实现直播流的拉取、转发和管理。系统具有以下特点：

  • 支持多种流媒体协议（RTMP、HLS等）
  • 支持批量导入和管理
  • 支持分类管理
  • 支持远程推流
  • 支持实时监控和状态查看

2. 功能特点

2.1 基础功能

  • 流媒体添加、编辑、删除
  • 流媒体分类管理
  • 实时状态监控
  • 流量统计
  • 在线播放器

2.2 高级功能

  • 批量导入导出
  • 远程推流
  • 自动故障恢复
  • 系统状态监控

3. 安装部署

3.1 系统要求

  • Node.js 14.0 或以上版本
  • FFmpeg 4.0 或以上版本
  • 2GB以上内存
  • 10GB以上硬盘空间

3.2 安装步骤

# 克隆项目
git clone https://github.com/your-repo/stream-manager.git

# 安装依赖
cd stream-manager
npm install

# 启动服务
npm start
注意：首次启动时，系统会自动创建配置文件和必要的目录结构。

4. 基本使用

4.1 添加流

  1. 点击"管理工具"展开工具面板
  2. 在"添加流"区域填写以下信息：
    • 分类：流的分类名称（可选）
    • 流名称：给流起一个易识别的名称
    • 源地址：流的源地址（支持RTMP/HTTP/HTTPS）
    • 自定义ID：可选，留空将自动生成
  3. 点击"添加流"按钮完成添加

4.2 批量导入

系统支持批量导入流，格式如下：

新闻频道,#genre#
CCTV新闻,http://example.com/cctv-news
凤凰资讯,http://example.com/phoenix-news

体育频道,#genre#
CCTV5,http://example.com/cctv5
ESPN,http://example.com/espn

4.3 流管理

在流列表中，您可以对每个流进行以下操作：

  • 编辑：修改流的名称、地址等信息
  • 手动启动：手动启动流并推送到远程服务器
  • 停止：停止流的运行
  • 删除：从系统中删除流

4.4 导出功能

系统支持多种导出格式：

  • 导出M3U：导出本地HLS播放列表
  • 导出TXT：导出分类格式的文本文件
  • 导出远程M3U：导出远程推流的播放列表
  • 导出远程TXT：导出远程推流的分类文本
  • 导出源地址：导出所有流的原始地址

5. 高级功能

5.1 远程推流

系统支持将流推送到远程服务器：

  1. 在服务器配置中设置RTMP推流和拉流地址
  2. 选择需要推流的频道，点击"手动启动"
  3. 系统会自动将流推送到配置的远程服务器
注意：远程推流功能会消耗额外的带宽和系统资源。

5.2 服务器配置

在服务器配置面板中，您可以设置：

  • 服务器地址：系统的访问地址（auto表示自动获取）
  • 端口：系统的监听端口
  • RTMP推流地址：远程RTMP服务器的推流地址
  • RTMP拉流地址：远程RTMP服务器的拉流地址

5.3 系统监控

系统提供实时监控功能：

  • 总流数量
  • 活跃流数量
  • 系统运行时间
  • 流量统计（接收/发送）

6. 常见问题

Q: 为什么有些流无法播放？

A: 可能的原因：

  • 源地址已失效或不可访问
  • 网络连接不稳定
  • 流格式不受支持

Q: 系统支持哪些流媒体格式？

A: 系统支持以下格式：

  • RTMP
  • HTTP-FLV
  • HLS (m3u8)
  • RTSP

Q: 如何提高系统稳定性？

A: 建议采取以下措施：

  • 定期清理不使用的流
  • 监控系统资源使用情况
  • 使用高质量的源地址
  • 适当调整并发数量
